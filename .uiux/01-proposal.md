# nautli.ai 랜딩 UI/UX 1차 설계안 (디렉터 제안)

원칙: 절제된 개발자 톤. 다크 우선(prefers-color-scheme 존중), 모노스페이스는 코드/수치에만, 애니메이션은 미묘하게, 실측 수치만.

### 2. 공유카드 착지 `/c/[shareId]`

SNS에서 "Alice의 볼트 67점"카드 보고 온 사람의 목적지. 이 페이지는 카드 자체가 스타이며, 홈과 다른 감정(호기심 → 나도) 유발이 목표.

**레이아웃**
- v1: 공유된 카드 자체를 크게 확대 (OG 이미지와 픽셀 동일)
  - 상단: "Alice's vault checkup"
  - 미세 텍스트: "generated on Alice's machine · Alice's data never left her disk"
- v2: **"너의 볼트는 몇 점?"** CTA
  ```
  $ npm i -g nautli && nautli scan
  ```
  한 줄 복사
- v3: 홈으로의 자연스러운 링크 ("nautli에 대해 더 →")

SEO: `/c/*`가 각 카드별 로케일로 크롤링 대상. `og:image`는 정적 렌더.

### 3. 신조·프라이버시 `/manifesto`

세계관 심화. 진짜 신자를 만드는 페이지.

**구성** (긴 형식 에세이 톤, 스크롤 서사)
- 도입: "GPT-4는 이미 없다. Claude 3도 곧 없다."
- 문제: "매번 자기소개 반복 vs 매번 다른 답"
- 우리 답: "기억은 사람 것. 모델은 도구."
- 우리가 안 하는 것 3가지 (전송 없음 / 몰래 쓰기 없음 / 벤더락인 없음)
- 우리가 하는 것 3가지 (사실 단위 append / 소화 리뷰 카드 / 로컬 검색)
- **투명성 섹션**:
  - 우리가 아는 것: 익명 설치 카운트 (opt-in)
  - 우리가 모르는 것: 노트 내용 · 검색어 · AI 대화 (일방적)
- 하단: 소스코드/라이센스/Discord

### 4. 설치 가이드 `/install`

## 핵심 유저 플로우

**Flow A — HN/Twitter 링크 유입 개발자**
홈 히어로 → 신조 3초 파싱 → 신뢰 3종 스크롤 → `[증거]` 1개 클릭해 검증 → 검진 훅 데모 카드 → 커맨드 복사 → 터미널로 이탈 → 30초 후 첫 검진 결과 → Discord 조인

**Flow B — SNS 카드 보고 유입**
`/c/abc123` 착지 → 친구 카드 감상 → "너도?" CTA → 커맨드 복사 → (선택) 홈 이동해 신뢰 확인 → 설치

**Flow C — Obsidian 파워유저, "AI가 내 볼트 만지나" 검색 유입**
검색 → `/manifesto` (SEO 착지) → "우리가 안 하는 것 3가지" 읽음 → 안심 → 홈으로 이동 → 설치

세 플로우 모두 종착지는 터미널(설치). 웹은 "설치까지의 최단 신뢰경로"만 담당.

## 주요 인터랙션

1. **커맨드 복사** — 클릭 즉시 버튼 라벨을 `Copied ✓`로 인라인 치환, 2초 후 원복. 토스트/모달 없음(개발자 인지 부하 최소).
2. **증거 accordion** — 신뢰 3종 카드의 `[증거]`는 아래로 부드럽게 펼침(200ms). 외부 링크는 새 탭.
3. **데모 검진 카드 툴팁** — hover 시 각 지표에 tooltip ("모순 쌍이란: 같은 사실에 대해 시점 다른 두 노트").
4. **언어 전환** — nav 우측 셀렉터. URL prefix + hreflang + `<html lang>` 동기화. 로컬스토리지 캐시.
5. **히어로 터미널 마이크로 애니메이션** — 커맨드 아래에 `nautli scan` 실행 로그가 40자/초로 타이핑, 4초 후 정지. `prefers-reduced-motion` 존중.
6. **다크/라이트** — `prefers-color-scheme` 존중. nav에 수동 토글은 두지 않는다(선택지 늘리지 않음).

## 확신 못 하는 열린 질문

1. **데모 검진 카드의 진실성 vs 훅 세기** — 예시 카드에 `EXAMPLE` 워터마크를 붙이면 개발자 신뢰는 얻지만 감정 훅이 약해진다. 반대로 익명 통계(median 62pt 등)를 쓰면 훅이 강해지는데 옵트인 telemetry가 필요하고 "우리가 아는 것" 섹션과 미묘한 긴장. 어느 쪽을 선택할지, 아니면 제3의 방법(예: "당신의 실제 볼트를 이 페이지에서 미리보기" — 브라우저에서 로컬 스캔 후 서버 무전송 렌더)이 있는지?

2. **홈에 신조를 얼마나 담나** — 히어로 한 문장으로 압축하면 훅은 강하지만 "실제 세계관"이 궁금한 사용자가 즉시 이탈. Manifesto 링크 하나로 넘기기엔 CTR이 낮을 수 있고, 홈에 신조 섹션을 넣으면 검진 훅/설치 커맨드로 가는 스크롤 코스트가 길어짐. 홈에 신조를 얼마나 담고, 어느 지점에 배치할지?

3. **공유카드의 저장 형태 vs 프라이버시 신조** — `/c/[shareId]`가 성립하려면 카드 데이터가 어딘가에 있어야 한다. (a) URL 파라미터에 base64 인코딩(무서버, URL이 300자+) (b) 정적 CDN 옵트인 업로드(간단, 신조와 미묘한 긴장) (c) OG 이미지만 서버 생성 후 즉시 폐기, 페이지는 파라미터(중간지). 이 선택이 "우리가 안 하는 것 3가지"의 문구 자체를 좌우한다.
