# nautli 상단 앱셸 UX — 1차 디렉터 안

Orca 룩(무채색·7% 헤어라인·잉크 타이포·violet=AI 진행 전용) 위에서 "옵시디언스러운 조용한 크롬 + Linear/Raycast식 명령 진입"을 목표로 잡는다.

## 1. 화면 목록 & 핵심 요소

### 1.1 상단 앱바 (persistent, 높이 40px)
심리스 타이틀바 안에 요소를 얹는다. 좌측 80px는 트래픽라이트 안전영역으로 완전히 비운다.

레이아웃 (좌→우):
- **0~80px**: 빈 여백 (트래픽라이트)
- **80~200px · 위치 라벨**: 현재 섹션명 (`질문` / `기억` / `그래프` / `설정`) — 13px, 잉크 텍스처 60%. 사이드바 하이라이트와 중복이지만 사이드바 접힘·전체화면 대비.
- **중앙 (max-width 480px, flex-center)**: 검색 진입 pill
  - `기억 검색…    ⌘K` — 배경 3% 그레이, 헤어라인 7%, radius 8px, height 28px
  - hover 시 배경 5%, 클릭 or ⌘K → 팔레트 오픈
  - 옵시디언 상단 검색바보다 살짝 좁게 (앱셸에 부담 X)
- **우측 (자동)**: 리뷰 큐 진행 + 밤 소화 상태
  - `오늘 12 / 28` + 24px 얇은 링 (2px stroke, 잉크 40%로 채워짐) — 클릭 시 `질문` 섹션으로 이동
  - 소화 진행 중이면 링 대신 violet dot pulse + `정리 중` 툴팁 (SSOT의 violet=AI 진행 규칙 준수)
  - 리뷰 0장이면 링만 회색 완결 상태 (`오늘 완료` 툴팁), 숫자 숨김

바 하단에 7% 헤어라인 1px. 그 외 그림자·구분선 없음.

### 1.2 ⌘K 검색 팔레트 (오버레이)
Raycast·Linear 스타일 중앙 카드.

- 위치: 뷰포트 상단 22% 지점, width 640px, radius 12px, 배경 #16, 헤어라인 7%
- 백드롭: 배경 60% 어둡게 + 백드롭 클릭/ESC로 닫힘
- 구성:
  - **입력행** (56px): 좌측에 검색 아이콘, placeholder `기억·결정·사람·프로젝트 검색`
  - **결과 리스트** (max 8줄, 스크롤): `/api/memory?q=` 재사용
    - 각 행: 스니펫 1줄(잉크 90%, 하이라이트는 잉크 100%+underline) + 하단 메타 1줄 `scope·date·source` (10px, 잉크 50%)
    - 선택된 행: 배경 5%, 좌측 2px violet 인디케이터 **금지** (violet 오용) → 대신 잉크 100% 좌측 2px
  - **하단 힌트 바** (32px): `↑↓ 이동  ↵ 열기  ESC 닫기` — 10px, 잉크 40%
- 빈 상태: 최근 검색어 5개 (localStorage), placeholder는 `요즘 자주 찾는 것`

기존 백엔드만 씀. 신규 인덱스 없이 q= 그대로.

### 1.3 리뷰 큐 진행 표시의 상태 변주 (앱바 내)
- **대기중** (오늘 카드 있고 미착수): `오늘 0 / 28` + 회색 링
- **진행중**: `오늘 12 / 28` + 링 42% 채움
- **완료**: 링만 완결 상태, 숫자 페이드
- **소화중**: 링 자리에 violet dot pulse + 완료 시 `+7장 새 리뷰` 토스트 (앱바 아래 슬라이드다운, 3초 후 소멸)

## 2. 핵심 유저 플로우

**플로우 A — 아침 진입 (최빈)**
1. macOS 알림 or 메뉴바 뱃지 클릭 → 앱 포그라운드
2. 시선이 우측 상단 `오늘 0 / 28` 링에 착지 (또는 창 진입 즉시 자동 라우팅)
3. 링 클릭 → `질문` 섹션 · 첫 카드 포커스
4. 답변 진행하면 상단 링이 실시간으로 채워짐 (다른 섹션 이동해도 진행률 유지 → 이탈 후 복귀 유도)

**플로우 B — 기억 리콜**
1. 어디서든 ⌘K → 팔레트 오픈
2. 타이핑 (debounce 120ms) → 결과 리스트 갱신
3. ↓ ↑ 로 선택 → ↵ → 본문에 해당 memory 상세 페이지 로드, 팔레트 닫힘
4. ESC → 원래 섹션 유지

**플로우 C — 백그라운드 소화 인지**
1. 낮 동안 데몬이 임시 실행 (수동 트리거 or 개발 중 재소화)
2. 우측 상단 violet dot pulse → hover 시 `기억 정리 중… 42%`
3. 완료 시 슬라이드다운 토스트 `정리 완료 · +7장 리뷰` → 클릭 시 질문 섹션

## 3. 주요 인터랙션 (키보드 우선)

| 키 | 동작 |
|---|---|
| `⌘K` | 팔레트 열기/닫기 |
| `⌘.` | 사이드바 토글 (옵시디언 관행) |
| `↑ ↓` | 팔레트 결과 이동 |
| `↵` | 결과 열기 |
| `ESC` | 팔레트 / 오버레이 닫기 |
| `⌘1~5` | 사이드바 섹션 점프 (설정/그래프/질문/기억) |
| `⌘J` | 리뷰 큐로 점프 (진행 링 클릭과 동일) |
| `?` (섹션 내) | 해당 섹션 단축키 시트 |

마우스: 상단 진행 링·현재 위치 라벨은 hover 시 배경 5%, 클릭 가능함을 pointer로만 암시(외곽선·아이콘 X — 조용한 크롬 유지).

## 4. 빼거나 미룬 것 (왜)

- **탭 시스템·북마크·파일트리**: 스코프 밖 + nautli는 "카드에 답한다 / 검색해서 본다"가 주 여정이라 탭이 여러 개 살 이유가 없다. 옵시디언식 다중창은 노트 편집 워크플로에 최적이지 nautli의 리뷰 큐+리콜에는 오히려 방해.
- **상단 브레드크럼(2단계 이상)**: 계층이 얕고(섹션→항목) 사이드바가 이미 위치를 보여줌 → 위치 라벨 1단으로 충분.
- **알림 벨 아이콘**: 리뷰 큐 진행 링이 사실상 인박스 역할. 벨을 따로 두면 정보원 이중화.

## 5. 열린 질문 (토론 요청)

1. **상단 "현재 위치" 라벨이 정말 필요한가?** 사이드바 하이라이트와 중복이라 노이즈일 수 있다. 대안: 라벨 자리를 비우고 좌측을 완전 무음으로 둔다(옵시디언은 실제로 상단 좌측이 거의 비어 있음). 전체화면·사이드바 접힘 케이스가 얼마나 흔한지가 판단 기준.
2. **⌘K 팔레트에 "명령"도 넣을까 순수 "검색"만?** Raycast/Linear는 명령(섹션 이동·설정 열기·소화 지금 돌리기)까지 팔레트에서 처리. 이걸 넣으면 강력하지만 결과 리스트가 뒤섞여 검색 속도감이 떨어질 수 있다. 대안 A: 검색 전용. 대안 B: 첫 글자가 `>`일 때만 명령 모드 (Linear 방식). 유저가 개발자라 명령 팔레트를 이미 익숙해할 확률이 높음.
3. **리뷰 큐 진행 링이 0/0(리뷰 없음)일 때 완전히 숨길까, 회색 완결 상태로 남길까?** 숨기면 앱바가 더 조용해지지만, "오늘 소화 결과가 정말 0인 건지 앱이 멈춘 건지" 불안 유발 가능. 밤 소화 마지막 실행 시각을 툴팁에라도 노출할지.
