# 타겟 유저 피드백 (codex 페르소나 — gemini CLI 사망으로 대체, 2026-07-16)

### 내가 이걸 쓰는 상황

금요일 밤, Cursor가 예전에 끝난 방식으로 코드를 고치고 Codex는 새 규칙을 따르는 바람에 같은 작업을 두 번 되돌린 상태다. 레딧에서 “내 볼트에 모순 47개 나옴”이라는 공유카드를 보고 들어온다. 광고 냄새가 나면 바로 닫을 생각으로 휴대폰에서 먼저 본다.

“AI를 바꿔도 기억은 남는다”는 말에는 바로 공감한다. 다만 설치는 업무용 맥에서 해야 하므로 링크만 저장해 둔다.

다음 날 맥에서 다시 열고 실제로 하려는 일은 단순하다.

1. 내 옵시디언 노트 4천 개를 건드리지 않고 읽을 수 있는지 확인한다.
2. Claude Code, Codex, Cursor가 정말 같은 규칙을 보게 되는지 확인한다.
3. 무엇이 밖으로 나가는지 소스와 명령으로 확인한다.
4. 괜찮으면 빈 테스트 폴더에서 먼저 실행한다.
5. 결과가 납득될 때만 실제 저장소와 옵시디언 폴더를 연결한다.

나는 첫 화면의 설치 명령을 바로 복사하지 않는다. GitHub에서 최근 변경 내역, 설치할 때 요구하는 권한, 실제 저장 파일 예시부터 찾는다.

### 써보니 불편한 점 (구체적으로)

- 가장 궁금한 “그래서 세 AI가 정확히 무엇을 같이 보게 되나?”가 바로 안 나온다. 대화를 전부 기억하는지, 내가 승인한 규칙만 전달하는지, 매번 어떤 글이 붙는지 모르겠다.

- “내용 무전송”과 “밤에 모순 탐지”가 함께 나오니 의심부터 든다. 내 컴퓨터 안에서 무엇이 판단하는지, 별도 AI가 필요한지, Claude나 OpenAI로 내용을 보내는지 설명이 없다.

- `0 bytes transmitted`는 너무 단정적이다. 설치 자체는 인터넷을 쓰고, 익명 설치 수집과 공유카드도 언급된다. “내 노트 내용은 nautli 서버로 보내지 않는다”처럼 범위를 정확히 말해야 믿는다.

- 공유카드의 숫자가 서버에 저장되는지도 불명확하다. “Alice의 데이터는 디스크를 떠나지 않았다”고 써놓고 웹 주소로 공유된다면 무엇이 올라갔는지 바로 의심한다.

- “볼트 67점”은 게임처럼 보여서 오히려 신뢰가 떨어진다. 왜 67점인지 기준도 없다. 오래된 노트와 새 노트의 차이를 전부 모순으로 잡으면 숫자만 요란할 가능성이 크다.

- “모순 47쌍”도 그 자체로는 쓸모가 없다. 예전에 PostgreSQL을 쓰겠다고 적었다가 나중에 SQLite로 바꾼 것은 오류가 아니라 기록이다. 이런 경우를 제대로 구분하는 실제 예시가 필요하다.

- “3만 노트, 30초”는 내 컴퓨터에서도 그런지 알 수 없다. 어떤 컴퓨터, 어떤 크기의 노트, 첫 실행인지 다시 실행한 것인지 없으면 광고 문구로 읽힌다.

- “SQLite + Markdown이 원본”이라는 표현은 이해가 안 된다. 둘의 내용이 다르면 어느 쪽이 진짜인지, 내가 Markdown을 직접 고치면 반영되는지 알고 싶다.

- 옵시디언 사용자인데 어떤 폴더와 파일을 읽는지 나오지 않는다. 첨부파일, 숨김 폴더, 개인 일기, 다른 플러그인 데이터까지 훑는지 걱정된다.

- 연결 과정에서 Claude Code·Codex·Cursor의 어떤 파일을 바꾸는지 모르겠다. 기존 설정을 덮는지, 제거하면 원래대로 돌아오는지도 설치 전에 확인하고 싶다.

- 공유 페이지의 `npm i -g nautli && nautli scan` 한 줄 실행은 싫다. 처음 보는 프로그램을 전체 컴퓨터에 설치하고 곧바로 내 파일까지 훑으라는 뜻처럼 보인다.

- 첫 화면의 타이핑 효과와 짧은 영상은 필요 없다. 나는 움직이는 로그보다 실제로 생성되는 파일 한 장을 보고 싶다.

- Manifesto의 “GPT-4는 이미 없다” 같은 문장은 과장처럼 느껴진다. 제품이 내 파일을 어떻게 다루는지보다 세계관을 먼저 설득하려 하면 뒤로 간다.

- Discord 가입이 설치 흐름의 끝인 것도 어색하다. 나는 커뮤니티에 들어가고 싶은 게 아니라, 문서만 보고 조용히 쓸 수 있기를 원한다.

### 진짜 필요한 것 / 빼도 되는 것

진짜 필요한 것:

- 설치 전에 볼 수 있는 실제 예시 하나. 세 AI에 같은 규칙을 저장하고, 다음 작업에서 각각 어떻게 읽으며, 규칙이 바뀌었을 때 어떤 리뷰가 뜨는지 처음부터 끝까지 보여줬으면 한다.

- “읽는 것, 쓰는 것, 밖으로 보내는 것”을 한눈에 볼 수 있는 짧은 표. 선택 수집과 공유 기능은 기본값이 꺼져 있는지도 함께 보여줘야 한다.

- 연결할 때 바뀌는 파일 목록과 제거 방법. 기존 설정 파일의 앞뒤 차이도 보고 싶다.

- 옵시디언 폴더는 처음에 읽기 전용으로 열 수 있어야 한다. 내가 고른 폴더만 읽고, 제외할 폴더와 파일을 지정할 수 있다는 설명이 필요하다.

- 빈 예제 폴더로 먼저 실행하는 명령. 실제 노트나 업무 저장소를 연결하기 전에 결과 형태와 속도를 확인하고 싶다.

- 점수가 아니라 실제 리뷰카드 2~3개. “진짜 모순”, “시간이 지나 바뀐 결정”, “단순 중복”을 각각 어떻게 보여주는지가 중요하다.

- “내용 무전송”을 내가 직접 확인하는 짧은 방법. 소스 파일 링크만 주기보다 실행 중 연결이 생기는지 확인하는 명령을 그대로 복사할 수 있으면 좋겠다.

- 지원하는 파일과 운영체제, 필요한 프로그램, 첫 실행 시간, 밤에 자동으로 실행되는 방식에 대한 정확한 설명.

빼도 되는 것:

- 볼트 점수와 점수 경쟁 느낌의 공유카드. 문제 개수와 실제 예시만으로 충분하다.
- 히어로의 타이핑 효과와 단계별 짧은 영상.
- 긴 Manifesto와 강한 신조 문구. 첫 화면에서는 한두 문장이면 된다.
- X, Discord, 후원 링크를 여러 곳에서 강조하는 것.
- 첫 방문자에게 바로 전체 설치를 권하는 한 줄 명령.

내가 설치하게 만드는 것은 멋진 문장이나 67점 카드가 아니다. “어떤 파일을 읽고, 무엇을 바꾸며, 무엇도 보내지 않는지”를 설치 전에 구체적으로 확인할 수 있다는 점이다.
