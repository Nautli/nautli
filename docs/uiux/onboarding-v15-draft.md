# 온보딩 강화 v1.5 디렉터 초안 (2026-07-14) — Orca 구조 번안

> 배경: Orca 온보딩 분석(Evanwiki/신사업/orca-분석-nautli-이식후보.md + 스크린샷 6장).
> 훔치는 건 구조 3개: (1) 자동 감지가 첫인상 (2) 위저드 최소+상주 체크리스트로 활성화 지속 (3) 개인 숫자로 설득.
> 그대로 베끼지 않는 것: 테마 선택(불필요), 폰 앱 카드(제품 없음), Yolo 옵션(해당 없음).

## A. 신규: 감지 히어로 "당신의 AI들" (Setup 탭 최상단)

**목적**: 설치 직후 빈 DB인데도 첫 화면이 유저 자신의 데이터로 차 보이게. 상실 프레임으로 문제를 본인 숫자로 증명.

**서버 스캐너** `src/onboard/scan.js` (신규):
- AI CLI 감지: PATH에서 `claude`/`codex`/`cursor`/`gemini` 존재 여부 (`execFile <bin> --version`, 각 2s 타임아웃, 실패=미설치).
- 세션 스토어 통계 (콘텐츠 비보존 — 파일 메타+타임스탬프만):
  - Claude: `~/.claude/projects/**/*.jsonl` — 파일 수, 최근 30일 필터(mtime), 각 파일 첫줄+마지막줄의 timestamp만 읽어 세션 시간합(라인 전체 파싱 금지, 첫/끝 라인만. 64KB 캡 스트림).
  - Codex: `~/.codex/sessions/**/*.jsonl` 동일.
  - Cursor/Gemini: 존재 감지만(통계 v2).
- nautli 기억 수: store.query() active count.
- 결과 스키마: `{ agents: [{id, name, cli: bool, sessions30d: int|null, hours30d: number|null}], totals: {sessions, hours}, remembered: int, scanned_at }`
- 캐시: `~/.nautli/scan.json`에 저장, 24h TTL, Setup 탭 진입 시 stale이면 백그라운드 재스캔. API: `GET /api/scan` (캐시 반환+`?refresh=1`).
- 보안·성능: 홈 하위만, symlink 미추적, 파일 3,000개 캡(초과 시 "3,000+"), 스캔 총 5s 예산 초과 시 부분 결과+`partial:true`.

**UI (Setup 최상단 카드)**:
- 헤드라인: "감지된 AI {N}개 · 최근 30일 대화 {X}시간" (통계 없으면 "감지된 AI {N}개")
- 서브라인(상실 프레임): "그중 nautli에 기억된 건 {remembered}개예요." — remembered=0이면 "0개예요. 오늘부터 바꿔요."
- 에이전트 행: 아이콘+이름+상태칩(연결됨(MCP 등록)/설치됨/미설치). Claude 행의 [연결] 버튼 = 기존 스텝2와 동일 동작(중복 UI 아님 — 아래 스텝퍼의 스텝2를 이 카드가 대체·흡수).
- 정직성: 시간은 "약 {X}시간" 표기, hover에 "세션 파일의 시작·끝 시각으로 추정" 툴팁. 감지 0개면 카드 대신 기존 흐름.
- 프라이버시 고지 1줄: "스캔은 파일 개수와 시각만 읽어요. 대화 내용은 읽지 않아요."

## B. 재편: 상주 체크리스트 "다음 할 일" (필수 4개 완료 후)

현재: 필수 4 완료 → 배너+연속성 히어로+선택 섹션이 병렬 나열. 재편:
- 필수 4개 완료 시 Setup 탭이 "다음 할 일" 체크리스트 모드로 전환 (진행도 {done}/{total} 표기, Orca 사이드바 체크리스트의 탭 내 번안 — 새 사이드바 만들지 않음, 차체 유지):
  1. 첫 진짜 기억 (기존 연속성 스텝, 상태 그대로: 완료/건너뜀)
  2. 내 기억 건강검진 (기존 검진, done/dismissed 반영)
  3. 두 번째 AI 연결 (기존 Cursor 스니펫 행 승격. 감지 카드에서 설치됨인데 미연결이면 이름 명시: "Cursor 연결")
  4. 공유 카드 만들기 (기존 공유카드 진입점 흡수)
  5. GitHub 별 하나 (아래 C)
- 각 행 = 상태아이콘+제목+1줄 설명+버튼 1개. 미완 첫 행만 primary(주황 1개 원칙 유지).
- 전부 완료 시: "다 됐어요. 이제 nautli는 알아서 굴러가요" 종결 배너(체크리스트 접힘, quiet 링크로 재열람).

## C. 신규: 성공 순간 star 요청 (Orca star-nag 번안)

- 트리거: 연속성 스텝B(recall 시연) 성공 직후 토스트 1회: "방금 그 기억, 마음에 들면 별 하나 주세요" [GitHub 열기] [나중에].
- 규칙: 평생 1회 노출(`~/.nautli/config.json`에 `star_nag_shown_at`), [나중에]면 체크리스트 5번 행으로만 남김. 실패/에러 화면에서 절대 노출 금지.

## D. 잔손질

- 위저드 스텝에 키보드: Enter=현재 primary 버튼 (전역 1개라 안전). 건너뛰기 링크 스타일 통일.
- 검진 시작 모달의 후보 감지 결과에 A 스캐너 결과 재사용(중복 스캔 금지).

## 비스코프 (명시 거절)

- 사이드바 신설, 테마, 폰/원격, 자동 캡처(별도 스파이크), Codex MCP 자동 등록(어댑터 확장은 별건 — 감지·표시까지만).

## 성공 기준 (착수 앵커)

1. 격리 홈 + 모의 세션 파일 셋(claude 12개·codex 3개)에서 스캔 API가 스키마대로 반환, 5s 예산, 콘텐츠 비보존 (25)
2. 신규유저 e2e: 첫 진입에 감지 카드 렌더(숫자 정확)→필수 4 완료→체크리스트 모드 전환→star 토스트 1회성 (25)
3. 기존 테스트 60/60 유지 + 신규(스캐너 엣지: 빈 홈/거대 파일/깨진 jsonl/3000캡) (20)
4. 비주얼 QA: 감지 카드 4상태(0개/통계없음/정상/partial)+체크리스트 2상태 — 겹침·잘림·상태누수 0 (20)
5. 카피: 줄표 금지·사람말·과장 금지(추정치 "약" 표기) (10)

## 상태머신·스키마 요약 (스펙 완결성 프리플라이트)

- scan.json: `{version:1, scanned_at, partial, agents[], totals{}, remembered}` — 버전 불일치 시 재스캔.
- 체크리스트 상태 = 기존 각 기능의 상태에서 파생(신규 저장 없음). star만 config에 `star_nag_shown_at`, `star_done`(수동 완료 클릭).
- 에러: 스캔 실패는 카드 비표시(콘솔만) — 온보딩 블로커 금지. E_SCAN_TIMEOUT은 partial로 흡수.
