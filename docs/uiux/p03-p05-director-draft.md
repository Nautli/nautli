## P0-3 온보딩 연속성 검증 스텝 ("진짜 첫 왕복")
문제: 온보딩 정점이 "볼트 점수"인데 제품의 획득 소구는 "크로스AI 연속성". 유저가 연속성을 한 번도 경험 못 하고 온보딩이 끝남.
제안:
1. Setup 필수 4단계 완료 시 히어로 카드 "이제 진짜로 해보죠" 노출 (기존 체험소화 히어로 자리 대체, 합성 샘플 체험은 선택 섹션으로 강등).
2. 스텝 A: [복사] 버튼 — "Remember this: my test phrase is {랜덤단어2}" → 유저가 Claude Code에 붙여넣음. 대시보드는 이벤트 로그 폴링으로 해당 fact remember 감지 → "Claude가 저장했어요" 체크.
3. 스텝 B: [복사] 버튼 — "What is my test phrase?" → 다른 툴(Cursor 연결 시 Cursor, 아니면 새 Claude 세션 안내)에서 물어보게 함. recall 이벤트 감지 → 완료 배너 "방금 두 세션이 같은 뇌를 썼어요".
4. 감지 타임아웃(120s) 시 폴백: "직접 확인했어요" 수동 버튼 + doctor 링크. 감지 실패가 온보딩을 영구 블록하면 안 됨.
열린 질문: 랜덤 문구 fact가 기억에 남는 쓰레기가 되는 문제(완료 시 자동 archive?), MCP recall 이벤트를 어떻게 대시보드가 아는가(이벤트 로그에 recall 기록 필요 여부).

## P0-4 검진 리포트 공유 카드
문제: "네 볼트 몇 점?"이 K루프 후보인데 리포트가 로컬 모달 종착점. raw 공유 시 개인정보 사고 경로.
제안:
1. 리포트 카드에 [공유 카드 만들기] → 미리보기 모달 → PNG 다운로드(캔버스 로컬 렌더, 서버 무).
2. 카드 내용(고정, 원문 옵션 자체를 안 둠): 점수/100 + 중복 N·모순 N·junk% + "sampled 40 notes" 명시 라벨 + nautli 브랜드 + CTA "How healthy is your vault? npx nautli dashboard". claim·파일경로·프로젝트명 절대 미포함.
3. 옆에 X/Threads용 텍스트 스니펫 [복사] 버튼.
열린 질문: 카드 언어(영/한 토글?), 점수 낮을 때 공유 동기(점수 프레임 대신 "발견 N건" 프레임?).

## P0-5 검진 하드닝 (UX 접점만)
1. 검진 시작 모달에서 Claude CLI 로그인 사전검사(doctor 로직 재사용) — 실패 시 시작 비활성+사유. 10분 뒤 부분실패 차단.
2. 표본을 경로순 → 시드 고정 셔플(재현성 유지). 리포트/공유카드에 표본 크기 상시 라벨.
3. 시작 모달에 제외 폴더 입력(콤마 경로) + 스캔 대상 파일 수 사전 표시(--estimate 재사용).
